# Mod Curation Policy

Every mod must deepen the same industrial sandbox instead of forking players into a parallel game.

Approve a mod when it:

- strengthens processing, power, logistics, or progression clarity
- fills a genuine capability gap
- integrates through shared tags and recipes
- can be tuned through config or light scripting

Reject a mod when it:

- adds an independent resource ladder with weak integration
- trivializes logistics, storage, or processing too early
- expects bespoke custom content just to feel complete
- overlaps a stronger incumbent without adding systemic value
